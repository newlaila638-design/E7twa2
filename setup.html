🎯 إعداد تطبيق احتواء

اتبع هذه الخطوات بعناية لإعداد المشروع بشكل صحيح

1

إنشاء مشروع Firebase

أولاً، ستحتاج إلى إنشاء مشروع Firebase:

  1. اذهب إلى Firebase Console
  2. انقر على "إنشاء مشروع جديد"
  3. أدخل اسم المشروع: احتواء (أو أي اسم تفضله)
  4. اتبع الخطوات حتى يتم إنشاء المشروع
✅ نصيحة: تذكر اسم المشروع، ستحتاجه لاحقاً
2

الحصول على بيانات الاعتماد

الآن ستحتاج إلى نسخ بيانات اعتماد مشروعك:

  1. من لوحة Firebase، اضغط على ⚙️ الإعدادات (رمز المسننات)
  2. اختر إعدادات المشروع
  3. ابحث عن Web API Key
  4. انسخ جميع البيانات التالية:
apiKey authDomain projectId storageBucket messagingSenderId appId
3

تفعيل Authentication

تحتاج إلى تفعيل خدمة المصادقة:

  1. في Firebase Console، انقر على Authentication
  2. اضغط على البدء
  3. اختر Email/Password
  4. فعّل خيار "البريد الإلكتروني وكلمة المرور"
  5. انقر على حفظ
⚠️ تنبيه: تأكد من تفعيل هذا الخيار وإلا لن تتمكن من تسجيل الدخول
4

إنشاء Firestore Database

الآن ستنشئ قاعدة بيانات Firestore:

  1. اضغط على Firestore Database من القائمة الجانبية
  2. انقر على إنشاء قاعدة بيانات
  3. اختر وضع التطوير (للاختبار)
  4. انقر تالي واختر موقعك الجغرافي
  5. انقر إنشاء
✅ ملاحظة: وضع التطوير يرفع الأمان لاحقاً قبل الإطلاق
5

تحديث firebase-config.js

أدخل بيانات Firebase في الملف firebase-config.js:

  1. افتح ملف firebase-config.js من المشروع
  2. استبدل البيانات "YOUR_..." ببيانات مشروعك
const firebaseConfig = { apiKey: "YOUR_API_KEY_HERE", authDomain: "YOUR_PROJECT.firebaseapp.com", projectId: "YOUR_PROJECT_ID", storageBucket: "YOUR_PROJECT.appspot.com", messagingSenderId: "YOUR_MESSAGING_SENDER_ID", appId: "YOUR_APP_ID" };
6

تشغيل الخادم المحلي

الآن تابع هذه الخطوات لتشغيل التطبيق:

  1. افتح Command Prompt / PowerShell
  2. اذهب لمجلد المشروع:
    cd "New folder (7)"
  3. شغل الخادم:
    npm start
  4. انتظر الرسالة: ✅ الخادم يعمل على: http://localhost:3000
✅ نجح! الخادم يعمل الآن
7

فتح التطبيق

افتح متصفحك وانتقل إلى:

http://localhost:3000

ستجد الصفحات التالية:

  • الصفحة الرئيسية - عرض معلومات التطبيق
  • تسجيل الدخول - دخول المستخدمين
  • إنشاء حساب - تسجيل مستخدم جديد
  • لوحة التحكم - عرض جميع المستخدمين والإحصائيات
8

اختبار التطبيق

الآن اختبر التطبيق بإتباع هذه النقاط:

9

تأمين البيانات

قبل الإطلاق النهائي، قم بتوفير قواعد الأمان:

  1. في Firestore، انقر على القواعس
  2. استبدل القواعس الافتراضية بـ:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /users/{userId} { allow read: if request.auth.uid == userId; allow create: if request.auth.uid == userId; } } }
⚠️ مهم جداً: لا تستخدم وضع التطوير في الإنتاج! قم بتطبيق قواعس أمان شاملة
اذهب إلى التطبيق Firebase Console قراءة الوثائق
